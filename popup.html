About WebAudit

What is WebAudit?

WebAudit analyzes webpages for security vulnerabilities, code quality, and best practices in real-time. All analysis runs locally in your browser - no data is sent anywhere.

Understanding Your Score

  • 90-100 (Safe) - Page meets security & best practice standards
  • 70-89 (Moderate) - Minor issues found, recommended fixes available
  • Below 70 (High Risk) - Significant security or quality issues detected

Score Categories Explained

  • Security (50%) - HTTPS, CSP headers, SRI, X-Frame-Options, cookie security, DOM XSS protection
  • Structure (25%) - Alt text, meta tags, viewport, inline scripts, deprecated HTML
  • Code Quality (25%) - Accessibility (ARIA), performance (render blocking), lazy loading

How to Use

  1. Click Analyze Page to scan the current tab
  2. View your overall score - click for detailed breakdown
  3. Check category scores by clicking each row
  4. Review issues list - click any issue for full explanation
  5. Click Report for comprehensive analysis

Security Checks Performed

  • HTTPS - Ensures site uses encrypted connection
  • CSP (Content Security Policy) - Prevents XSS attacks
  • SRI (Subresource Integrity) - Verifies CDN scripts haven't been tampered
  • X-Frame-Options - Prevents clickjacking attacks
  • X-Content-Type-Options - Prevents MIME-sniffing attacks
  • Cookie Security - Checks for Secure and HttpOnly flags
  • DOM XSS - Detects dangerous sinks like eval(), innerHTML

Developer Best Practices

  • Alt Text - Images should have descriptive alt attributes
  • Form Labels - All inputs need associated labels for accessibility
  • Lazy Loading - Images should use loading="lazy" for performance
  • Render Blocking - Scripts should use defer/async attributes

WebAudit - All analysis runs locally. No data collected.
